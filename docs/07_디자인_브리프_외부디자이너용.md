# 습관 정원 — 디자인 브리프 (외부 디자이너용)

> 이 문서 하나로 외부 웹/프로덕트 디자이너가 맥락을 파악하고 작업을 시작할 수 있도록 정리한 핸드오프 브리프입니다.
> 동봉 문서: `06_UX_UI_감사_및_디자인_시스템_로드맵.md`(상세 감사·표준), `04_목업_mockups.html`(초기 목업).

---

## 1. 제품 한 줄 요약

**"매일의 작은 습관이 모여 꽃을 피우는" 정원 메타포 기반 습관 트래커.**
습관을 체크하면 포인트·경험치를 얻고, 정원의 식물이 자라며 레벨/희귀도가 올라간다. (옵션) 신앙 사용자를 위한 기도·경건 기능 포함.

## 2. 타깃 사용자 & 톤

- **사용자:** 한국어 사용자, 모바일 우선. 꾸준함을 게이미피케이션으로 동기부여받고 싶은 사람.
- **정서적 톤:** 차분하고 따뜻한 자연(흙·잎·꽃·하늘). 자극적이지 않고, 죄책감보다 **격려**. "완벽이 아니라 돌아오는 것"이 핵심 메시지.
- **레퍼런스 결:** Finch, Streaks, Things, Forest 같은 정돈되고 부드러운 습관/할일 앱. 화려함보다 **여백과 위계**.

## 3. 플랫폼 · 기술 제약 (디자인 시 반드시 고려)

- **모바일 PWA** (설치형). 데스크톱 전용 레이아웃 불필요.
- **콘텐츠 폭 최대 480px**, 가운데 정렬. 하단 **탭바 내비게이션**(고정).
- iOS/Android **safe-area**(노치·제스처바) 패딩 필요.
- **다크 모드 현재 없음** (라이트 단일). 추후 확장 가능성은 열어둘 것.
- 프레임워크: React 18 + Vite + **Tailwind CSS** + **Radix UI** + framer-motion + recharts.
  - → 디자인은 **토큰/유틸로 환원 가능한 형태**가 이상적(임의값 최소화). Figma 변수 ↔ CSS 변수 매핑을 권장.
- 폰트: **Pretendard**(한국어 가변 폰트). 교체 시 한글 가독성·웹폰트 용량 고려.

## 4. 현재 디자인 시스템 (이미 구축됨 — 정련 대상)

### 4.1 컬러 토큰

| 역할 | 변수 | 값 |
|---|---|---|
| 배경 | `--bg-base` / `--bg-surface` | `#F4F6EE` / `#FFFFFF` |
| 글자 | `--fg-primary` / `--fg-muted` / `--fg-faint` | `#2A2E27` / `#6B7164` / `#9AA08F` |
| 잎(주색) | `--leaf` / `--leaf-soft` | `#4F7A37` / `#E7F0DD` |
| 흙 | `--soil` | `#8A6E4B` |
| 꽃(강조) | `--bloom` / `--bloom-soft` | `#A85D0B` / `#FAEEDA` |
| 하늘(정보) | `--sky` / `--sky-soft` | `#1F5FA5` / `#E6F1FB` |
| 위험 | `--danger` / `--danger-soft` | `#B4402E` / `#F7E4DF` |
| 경계 | `--border` / `--border-soft` | `#DDE5D0` / `#EAF0E0` |
| 정원 씬 | `--garden-sky-top/bottom`, `--garden-sun`, `--garden-soil-*` | (일러스트 배경) |

radius: 6/10/16/full · shadow: sm/md · easing: `cubic-bezier(.22,1,.36,1)` · duration: 140/220/380ms.

### 4.2 타이포 스케일(제안 정착값)

24(display) / 18(heading) / 16(title) / 14(body) / 12(label). 본문 14↓·메타 12↓ 지양.

### 4.3 모션

식물 흔들림, 입자 드리프트, 초월 등급 오로라/꽃잎/별빛, 레벨업 배지 burst 등 **정원 세계관 전용 모션**이 자산. `prefers-reduced-motion` 대응 필수.

## 6. 해결하고 싶은 핵심 문제 (우선순위 상)

1. **'오늘' 홈 과부하 & 동시 강조 남발** → 위계·절제.
2. **일관성 부족** — 같은 역할 요소(버튼·카드·뱃지·토글)가 화면마다 미세하게 다름.
3. **타이포 스케일 난립**(9~11px 임의값 다수, 가독성 저하).
4. **아이콘 언어 혼용**(lucide 라인 아이콘 + 이모지).
5. **빈/로딩/에러 상태 부재**(데이터가 "툭" 나타남).
6. (시스템 일탈) OS 네이티브 다이얼로그·off-palette red·인라인 hex → 코드 측에서 정리 중.

## 7. 목표 / 비목표

**목표**
- 정원 세계관을 유지한 채 **차분하고 정돈된** 인상.
- 명확한 **타이포·간격·컬러 스케일**과 **컴포넌트 시안**(버튼/카드/행/뱃지/입력/탭/다이얼로그/빈·로딩·에러).
- '오늘' 홈을 포함한 핵심 3~4개 화면의 **하이파이 시안**.
- Figma 토큰 ↔ 코드 토큰 매핑(개발 이관 용이성).

**비목표**
- 브랜드/로고 전면 리브랜딩(현 아이덴티티 유지).
- 정원 일러스트/식물 아트의 전면 재제작(정련은 환영, 교체는 범위 외).
- 데스크톱 전용 반응형.
- 다크 모드(향후 과제로만 고려).

## 8. 기대 산출물 (디자이너 → 우리)

1. **디자인 토큰 시트**: 컬러/타이포/간격/radius/그림자/모션 — 코드 변수명과 1:1 매핑.
2. **컴포넌트 라이브러리(Figma)**: Button(변형), Card, ListRow, Badge, Switch/Toggle, Input/Slider, Tabs, Dialog, EmptyState, Skeleton, Toast.
3. **핵심 화면 하이파이**: 오늘(홈), 습관, 정원, 더보기(설정) — 각 화면 빈/로딩/채워진 상태.
4. **아이콘 가이드**: lucide 기반 사용 규칙 + 필요한 커스텀 SVG 목록.
5. **상태/마이크로카피 가이드**(선택): 격려 톤 문구 톤앤매너.

## 9. 협업 메모

- 가능하면 **Tailwind 친화적**(토큰·8pt 그리드·표준 radius)으로 설계 → 개발 이관 시 임의값/일탈 최소화.
- 한국어 텍스트 길이(조사·줄바꿈) 기준으로 컴포넌트 여백 검증 요망.
- 접근성: 본문 대비 AA, 터치 타깃 44px↑, reduced-motion 대체.
