01디자인 철학 & 무드

"앱을 쓸 때의 느낌"을 결정하는 방향. 기획서 §12에서 확정된 디자인 원칙입니다.

  • 자연스럽고 차분한 유기적 톤 — 식물·정원 메타포. 번쩍이는 네온 같은 과한 게임 UI는 피합니다.
  • 부드러운 그린·흙·햇살 + 절제된 강조 — 연한 자연광 배경, 포인트는 햇살색으로만 짚어줍니다.
  • 모바일 우선 — 380px 기준(412px까지 확장), 큰 터치 타깃, 1~2분 안에 끝나는 입력 흐름.
  • 시듦은 '경고'가 아니다 — 빨강 대신 차분한 베이지(--wither)로, 언제나 회복 가능성을 함께 제시합니다.
  • 라이트 고정 — 다크 모드는 MVP 범위 밖. 토큰이 CSS 변수라 추후 분기는 가능합니다.

04보이스 & 카피 톤 (앱의 감성)

이 앱의 "느낌"은 카피에서 나옵니다. 아래 문구는 모두 실제 코드·기획서에서 추출했습니다.

작성 원칙 — 균형형 톤

  • 과장된 칭찬·공허한 위로·이모지 남발·느낌표를 지양합니다.
  • 정량 데이터에 근거한 객관적 진술을 우선 — 시간·점수·횟수는 숫자로.
  • 진전·연속성(스트릭, 추세 개선)은 사실에 근거해 한 줄로 인정하고 다음 행동으로 잇습니다(momentum).
  • 실패·시듦에도 회복 가능성을 항상 함께 제시합니다.

AI 코치 시스템 프롬프트 (발췌)

당신은 본인 1인의 일일 데이터를 분석하는 코치다.
한국어로 응답한다. 정량 데이터에 근거한
객관적 진술을 우선한다.
과장된 칭찬·공허한 위로·이모지·느낌표는
쓰지 않는다. 단, 진전과 연속성은 사실에
근거해 한 줄로 인정하고, 다음 행동을
이어가도록 동기를 부여한다(momentum).

코치 카드 예시 톤

사실(숫자) → 연속성 인정 → 구체적 다음 행동. 느낌표·이모지 없음.

06모션 & 인터랙션 감성
