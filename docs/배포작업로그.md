# 작업 로그 — GitHub 푸시 기반 자동 배포 구성

- **프로젝트**: 습관 정원 (Habit Garden) — Firebase 기반 개인용 PWA
- **작업일**: 2026-05-21
- **목표**: 코드를 GitHub에 올리고, `main` 브랜치 푸시 시 Firebase로 자동 배포(CI/CD)되도록 구성

---

## 1. 사전 점검

작업 시작 전 프로젝트 상태를 확인했다.

- pnpm 모노레포 구조 (`apps/web`, `functions`, `shared`)
- Git 저장소가 아직 아님 (`.git` 없음)
- `.gitignore` 없음 → `node_modules`, 빌드 산출물 등이 그대로 커밋될 위험
- 배포 방식은 `deploy.bat`(로컬에서 `firebase deploy`) 수동 방식
- `apps/web/.env`의 Firebase 설정값은 클라이언트 공개용이라 커밋해도 안전
- Gemini API 키는 코드가 아닌 Firebase Functions Secret에 저장되어 있어 노출 위험 없음
- Firestore/Storage 보안 규칙이 특정 본인 계정(uid 1개)만 허용하도록 잠겨 있어, 공개 저장소로 올려도 데이터는 안전

## 2. 생성한 파일

### `.gitignore`
`node_modules`, 빌드 산출물(`dist`, `lib`), 로그, 서비스 계정 키, `planner_src.zip` 등을 제외.
단, `apps/web/.env`는 CI 빌드에 필요한 공개 설정값이라 의도적으로 추적하도록 예외 처리.

### `.github/workflows/firebase-deploy.yml`
`main` 푸시(및 수동 실행) 시 다음을 수행:
pnpm/Node 20 설정 → 의존성 설치 → 웹앱 빌드 → Functions 빌드 → 서비스 계정 자격증명 준비 → `firebase deploy`(hosting, functions, firestore, storage 전체).

검증: 가상의 git add로 시뮬레이션하여 `node_modules`·`zip`은 제외되고 소스·워크플로는 정상 추적됨을 확인.

## 3. 1차 배포 실패 → 원인 분석 및 수정

첫 푸시 후 CI가 **Functions의 TypeScript 빌드 단계에서 실패** (에러 10개).

| 원인 | 내용 | 조치 |
|------|------|------|
| firebase-functions 버전 불일치 | 코드는 v1 방식(`functions.region(...)`)인데 락파일 부재로 CI가 최신 v6를 설치 → 루트 export에 `region` 없음 | `awardEngine.ts`, `backup.ts`, `dailyReset.ts`, `feedback.ts`의 import를 `firebase-functions/v1`로 변경 |
| `index.ts` 구문 잘림 | 마지막 줄이 `export { monthlyBackup }`에서 끊겨 `from './backup';` 누락 (GitHub은 에러 10개까지만 표시되어 가려짐) | 구문 완성 |
| 산출물 경로 불일치 | tsconfig가 `../shared`까지 컴파일 → 산출물이 `lib/functions/src/index.js`로 중첩되는데 `package.json`의 `main`은 `lib/index.js`를 가리킴 | `main`을 `lib/functions/src/index.js`로 수정 |

**검증**: CI와 동일한 환경(firebase-functions 6.6.0)에서 `tsc` 빌드가 에러 0개로 통과. 5개 함수(`awardEngine`, `dailyReset`, `generateFeedback`, `monthlyBackup`, `reflectionAward`) 모두 정상 export, `main`이 가리키는 파일 존재 확인.

## 4. 2차 배포 실패 → 원인 분석 및 수정

빌드 에러는 사라졌으나 **배포 단계에서 exit 1** (에러 1개).

- **원인**: GitHub 저장소에 `FIREBASE_SERVICE_ACCOUNT` 시크릿이 등록되어 있지 않음 → 빈 자격증명으로 인증 실패.
- **부가 수정**: 워크플로의 자격증명 파일 경로가 `$HOME` / `env.HOME`으로 어긋날 수 있어, `${{ github.workspace }}` 기반 고정 경로로 변경.

## 5. 다음 작업 (사용자 진행 필요)

배포 범위는 **전부 배포(웹 + 함수 + 규칙, Blaze 플랜)**로 결정.

1. Google Cloud Console에서 서비스 계정 `github-deployer` 생성, 역할 **편집자(Editor)** + **서비스 계정 사용자(Service Account User)** 부여
2. 해당 계정의 **JSON 키** 생성·다운로드
3. GitHub → Settings → Secrets and variables → Actions에 이름 `FIREBASE_SERVICE_ACCOUNT`로 JSON 전체 등록
4. 워크플로 수정분과 함께 커밋·푸시하여 재배포

## 6. 권장 후속 조치

- **`pnpm-lock.yaml` 커밋**: 이번 빌드 실패의 근본 원인이 락파일 부재였음. 로컬에서 `pnpm install` 후 생성되는 락파일을 커밋하면 로컬·CI 패키지 버전이 일치. 이후 워크플로의 `--frozen-lockfile`/캐시 옵션 활성화 가능.
- **`GEMINI_API_KEY`**: 함수 코드가 `process.env`로 읽으므로, AI 피드백 기능 동작을 위해 `firebase functions:secrets:set GEMINI_API_KEY` 설정 필요 (배포 자체는 막지 않음).
