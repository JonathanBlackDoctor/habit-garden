## 2. 평소 업데이트 방법 (가장 자주 쓰는 흐름)

코드를 고친 뒤, 로컬 폴더에서:

```bash
# 1) (선택) 빌드가 깨지지 않는지 먼저 확인
cd apps/web
npm install            # 처음 한 번만, 또는 의존성 바뀌었을 때
npx vite build         # 성공하면 OK (dist/ 생성)

# 2) 저장소 루트로 가서 커밋 & 푸시
cd ../..
git add -A
git commit -m "변경 내용 설명"
git push origin main
```

push 후 **GitHub → Actions 탭**에서 "Deploy to GitHub Pages" 워크플로가 초록불이 되면
1~2분 안에 라이브 사이트에 반영됩니다.

> ⚠️ 빌드 검증은 `npx vite build`를 권장합니다.
> (`npm run build`는 `tsc -b`까지 돌려서 타입 에러로 막힐 수 있음. 실제 배포 워크플로는 `vite build`만 사용.)

## 3. 절대 건드리면 안 되는 설정 (이거 바꾸면 사이트 깨짐)

1. **`apps/web/vite.config.ts` 의 `base: '/habit-garden/'`**
   - GitHub Pages는 `/habit-garden/` 하위 경로로 서빙됩니다. base를 지우면 JS/CSS가 404 납니다.
   - 저장소 이름을 바꾸면 이 값도 같이 바꿔야 합니다.

2. **`apps/web/src/App.tsx` 는 `HashRouter` 사용**
   - GitHub Pages는 SPA의 deep link 새로고침을 지원하지 않습니다.
   - `BrowserRouter`로 되돌리면 `/habits` 같은 주소 새로고침 시 404가 납니다. HashRouter 유지하세요. (주소가 `.../#/habits` 형태)

3. **Firebase 승인된 도메인**
   - `jonathanblackdoctor.github.io` 가 이미 등록되어 있어야 Google 로그인이 됩니다.
   - 위치: Firebase 콘솔 → Authentication → 설정 → 승인된 도메인. (이번에 추가 완료)

## 4. 저장소에 들어있는 워크플로 2개

- **`.github/workflows/pages.yml`** — `main` push 시 자동 실행. `apps/web` 빌드 후 GitHub Pages 배포. (현재 운영용)
- **`.github/workflows/firebase-deploy.yml`** — **수동 실행 전용**으로 바꿔둠.
  - 백엔드(Functions/Firestore 규칙 등)를 다시 배포해야 할 때만 Actions 탭에서 "Run workflow" 로 실행.
  - 실행하려면 저장소 Secrets에 `FIREBASE_SERVICE_ACCOUNT`(서비스 계정 JSON)가 설정되어 있어야 합니다.

GitHub Pages 저장소 설정: Settings → Pages → Source = **GitHub Actions** (이미 설정됨).

## 5. ⏳ 아직 안 올라간 작업: "기도(Prayers)" 기능

최근 작업 중이던 **기도 기능**은 라이브에 **아직 포함되지 않았습니다.**
로컬 작업 폴더의 관련 파일들이 중간에 잘려 깨져 있던 것을 **모두 복구**해 두었고,
로컬에서 `npx vite build` 성공까지 확인했습니다. 다만 자동화 환경에 git 자격증명이 없어 푸시하지 못했습니다.

apps/web/src/routes/Main.tsx              # 홈 카드 → /prayers 연결

apps/web/src/features/prayers/BulkParse.tsx        # (신규)

### 기도 기능을 라이브에 올리는 방법 (자격증명 있는 환경에서)
로컬 폴더는 이미 이 변경들이 다 반영된 상태입니다. 그대로 한 번에 올리면 됩니다:

```bash
cd "D:\조나단\플래너\planner_web - 복사본"
cd apps/web && npx vite build   # 빌드 성공 확인 (중요!)
cd ../..
git add -A
git commit -m "feat: 기도(Prayers) 기능 추가"
git push origin main
```

push 후 Actions에서 초록불 확인하면 기도 기능까지 라이브에 반영됩니다.

> 참고: 현재 `origin/main`에는 위 파일들의 "기도 미포함" 버전(App.tsx)만 올라가 있습니다.
> 로컬 버전을 push하면 기도 포함 버전으로 덮어써집니다(정상 동작).

## 7. 문제 해결 (Troubleshooting)

- **사이트가 빈 화면 / JS 404**: `vite.config.ts`의 `base`가 `/habit-garden/`인지 확인.
- **주소 새로고침 시 404**: 라우터가 HashRouter인지 확인 (주소에 `#` 있어야 정상).
- **Google 로그인 팝업에서 `auth/unauthorized-domain`**: Firebase 승인된 도메인에 github.io 주소 확인.
- **Actions 빌드 실패**: Actions 탭 → 실패한 run → 로그 확인. 보통 타입 에러 또는 import 누락. 로컬에서 `npx vite build`로 재현 후 수정.
- **배포는 성공인데 옛 화면**: 브라우저 캐시/서비스워커. 강력 새로고침(Ctrl+Shift+R) 또는 PWA 캐시 삭제.
- **로컬에서 git push 자격증명 오류**: 본인 PC에서 Git Credential Manager로 GitHub 로그인 후 재시도.
